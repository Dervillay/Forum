(route) POST /signIn

Route:
MethodPath
POST /signIn
Attempts to sign a user in, checks if an account with the name in the request body exists and checks if their encrypted password matches that saved in users. On Successful sign in, a token is generated and sent in the response.
Body Parameters:
Name Type Description
signInUsername String The user signing in's username
signInPassword String The user signing in's password
Response:
Name Type Description
status JSON Whether request was successful or unsuccessful
message JSON Details the result of the request
token JSON (on success) A JavaScript Web Token that expires in 24 hours
Response Code:
Type Description
200 if sign in is successful
404 if no user matching signInUsername can be found
409 if the user attempting to log in is already logged in elsewhere
403 if password is incorrect
500 if server encounters an error

(route) POST /signUp

Route:
MethodPath
POST /signUp
Attempts to add a new user to users with the details submitted in the request body, checking if the submitted email or username already exist in users. If successful, the submitted password is encrypted with 10 salt rounds, the user is added to users and a token is sent in the response to validate methods whilst the user is logged in.
Body Parameters:
Name Type Description
username String The user's username.
email String The user's email
password String The user's password
Response:
Name Type Description
status JSON Whether request was successful or unsuccessful
message JSON Details the result of the request
token JSON (on success) A JavaScript Web Token that expires in 24 hours
Response Code:
Type Description
200 if sign up is successful
409 if an account with username or email matching those submitted already exists
500 if server encounters an error
